5. The two-portal submission flow

The Defence Innovation Loan requires submission to two separate portals. Failure on either = rejection at pre-sift.

Portal A — UKDI Online Submission Service

  • URL: https://community.dasa.service.mod.uk/s/sign-in
  • Submit FIRST. Generates an "ACC..." submission number that goes into Portal B.
  • 6-step form (Service / Summary / Innovation details / Finance & delivery / Additional info / Review & submit).
  • All Desirability/Feasibility/Viability answers go into textboxes — attaching as PDFs = automatic pre-sift fail.
  • Image attachments only: .png, .jpg, .jpeg. Use these for the dashboard / decoy attacker map / system architecture diagram.
  • Reference list capped at 10.
  • Doc 02 contains paste-ready content for every textbox.

Portal B — Innovate UK Submission Portal (Qualtrics)

  • URL: https://innovateuk.eu.qualtrics.com/jfe/form/SV_9HSRU1dB0E2wg8S

6. Loan economics (Cycle 1 spec — important to understand)

This is not a standard commercial loan. Three distinctive features that drive the finance model:

  1. Two interest rates: 3.7% during project period (R&D + pre-commercialisation) on amount drawn; 7.4% during repayment period on amount outstanding. Plus an additional 3.7% accrued and deferred during project period — paid back during repayment.
  2. Quarterly drawdowns during project period. Not a lump sum; you draw down every 3 months as the project proceeds, against milestones agreed with the Innovate UK monitoring officer.
  3. Quarterly interest payments during project period. You pay 3.7% interest every 3 months on the amount you have actually drawn down (not the total committed). Defaulting on quarterly interest = covenant breach.

Cycle 1 structure (as proposed): 18 months project + 5 years repayment = 6.5 years total (within the 7-year cap). Compressed timeline reflects existing TRL 6 baseline (live decoy fabric already capturing real attacker traffic) and prior R&D investment.
